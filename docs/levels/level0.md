# Prolog
## Visual:
- Schwarzer Bildschirm.
- Nur Textfeld.
```Text
Stille.
Leere.
Kein Raum.
Kein Licht.
Kein Code.
Aber etwas ist erwacht.
Wenn du existieren willst,
musst du es ausdrücken.
```

# Task 1 – Der erste Gedanke
Der Editor öffnet sich.
Folgender Code ist bereits vorhanden:
```
public class Main {
    public static void main(String[] args) {
        System.out.println("...");
    }
}
```

## Story
- Die Struktur ist da.
- Doch der Gedanke fehlt.

## Anweisung
>Ändere den Inhalt der Anweisung so,
>dass die Konsole folgendes ausgibt:
>Hello World!

## Erwartete Lösung
Der Code soll am Ende so aussehen:
`System.out.println("Hello World!");`

## Hint
>In der Print-Anweisung wird eingegeben,
>was in der Konsole ausgegeben werden soll.
>Das "..." muss ersetzt werden.
 
## Bei Erfolg
### Visual:
- Eine schwebende Insel entsteht aus Nebel.
- In der Mitte: ein pulsierender Kern.
- Noch keine Details.
- Nur Form.
```Text
Signal erkannt.
Existenz bestätigt.
Eine Welt ist entstanden.
Doch sie kennt dich noch nicht.
```

# Task 2 – Deine Identität
Der Code bleibt geöffnet.
Er wird nun erweitert.

## Story
```
Die Insel reagiert auf deinen Code.
Doch ohne Namen bist du nur ein Echo.
Sag der Welt, wer du bist.
```

## Anweisung
>Gebe eine weitere Zeile aus,
>mit folgendem Inhalt:
>„Mein Name ist X“
>Ersetze X durch deinen eigenen Namen.

## Hint
>Du kannst mehrere Zeilen von Print-Anweisungen
>in einer Methode haben.
>Du kannst die bestehende Anweisung kopieren
>und abändern.

## Bei Erfolg
### Visual:
- Core Compiler Fragmente verbinden sich zunehmend
- Ein leuchtendes Fragment wird sichtbar und schwebt in den Kern.
- Der Kern pulsiert stärker.
- Die Insel ist stabil – aber leer.

# Abschluss des Prologs
```Text
Du existierst.
Du hast einen Namen.
Der Core Compiler reagiert auf dich.
Doch die Insel ist noch unstrukturiert.
Ohne gespeicherte Werte gibt es keine Ordnung.
Wenn du diese Welt aufbauen willst,
musst du lernen, Informationen zu speichern.
```
